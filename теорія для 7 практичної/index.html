🟢 Дизайн та імплементація RESTful API (Backend)

Побудова архітектурних прошарків та контролера завдань (TaskController) на Node.js та Express.

Імплементовані маршрути та архітектура

Контролер TaskController реалізує повноцінне API для ресурсу /api/tasks з валідацією, пагінацією та фільтрацією.

Метод Маршрут Опис дії
GET /api/tasks Отримання списку завдань (з фільтрацією, сортуванням, пагінацією)
POST /api/tasks Створення нового завдання з валідацією схеми та HATEOAS посиланнями
GET /api/tasks/:taskId Отримання конкретного завдання з перевіркою MongoDB ID
PUT /api/tasks/:taskId Повне оновлення (заміна) завдання новим станом ресурсу
PATCH /api/tasks/:taskId Часткове оновлення окремих полів завдання (наприклад, статусу)
DELETE /api/tasks/:taskId Повне видалення завдання з поверненням 204 No Content

Код валідації та схем (Приклад валідатора)

Використання express-validator для контролю вхідних параметрів та ідентифікаторів:

const { body, param, validationResult } = require('express-validator');

const validateTask = [
  body('title')
    .trim().notEmpty().withMessage('Title is required')
    .isLength({ max: 255 }),
  body('priority')
    .optional().isIn(['low', 'medium', 'high']),
  body('assigneeId')
    .optional().isMongoId().withMessage('Assignee ID must be valid'),
  (req, res, next) => {
    const errors = validationResult(req);
    if (!errors.isEmpty()) {
      return res.status(400).json({ success: false, errors: errors.array() });
    }
    next();
  }
];

Повідомлення системи доступності

Це модальне вікно розроблено з повною підтримкою доступності WAI-ARIA:

  • Фокус автоматично переноситься на модальне вікно під час відкриття.
  • Натискання клавіші Escape закриває вікно.
  • Кнопка закриття має чітку текстову мітку aria-label.
  • За межі модального вікна фокус не виходить (Focus Trap).
